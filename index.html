📄 포트폴리오 요구사항 정의서

1. 프로젝트 개요

프로젝트 명: 개인 포트폴리오 웹사이트 제작

목적: 이력, 기술, 프로젝트 등을 정리해 채용·의뢰에 활용

형태: 반응형 정적 웹사이트

2. 주요 이해관계자 및 사용자

  • 이해관계자: 개발자 본인, 디자이너
  • 사용자: 채용 담당자, 클라이언트, 블로그 독자

3. 기능 명세서

  • 자기소개
  • 이력 및 경력
  • 기술 스택 소개
  • 프로젝트 목록 및 상세
  • 연락 폼
  • 소셜 링크
  • 블로그 (옵션)
  • 다크모드 전환

4. 비기능 요구사항

  • 빠른 로딩
  • 스팸 방지
  • 모바일 최적화
  • 유지보수 용이성
  • 브라우저 호환

5. 시스템 범위

포함: 정적 웹 개발, 배포, 기본 UI 디자인

제외: 인증 시스템, 서버사이드 처리

6. 기술 스택 정의

  • HTML, CSS, JavaScript 또는 React
  • 디자인: Figma, TailwindCSS
  • 배포: GitHub Pages, Netlify
  • 버전관리: Git

7. 프로젝트 목표 요약
